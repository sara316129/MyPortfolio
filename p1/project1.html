How It Was Built

Step-by-Step Process

01

Data Collection

Downloaded the Superstore Sales dataset from Kaggle - a real-world retail dataset containing 9,994 rows across orders, customers, products, regions, and returns. Imported into SQL Server (SSMS) as the starting point.

02

Data Cleaning & Preparation (SQL)

Used SQL queries to clean and prepare the data before loading into Power BI:

  • Removed null values and duplicate rows
  • Fixed incorrect data types (dates, numeric fields)
  • Created derived columns: Year, Month, Profit Flag
  • Joined Orders, Returns, and People tables with LEFT JOINs
  • Used COALESCE to handle missing return status values
  • Rounded sales figures with ROUND() for consistency
SQL — Data Preparation Query 
SELECT
  o.Order_ID,
  o.Order_Date,
  o.Ship_Date,
  o.Ship_Mode,
  o.Customer_ID,
  o.Customer_Name,
  o.Segment,
  o.Country,
  o.City,
  o.State,
  o.Postal_Code,
  o.Region,
  p.Person AS Region_Manager,
  o.Product_ID,
  o.Category,
  o.Sub_Category,
  o.Product_Name,
  o.Quantity,
  ROUND(o.Sales, 2)         AS Sales,
  ROUND(o.Profit, 2)        AS Profit,
  ROUND(o.Discount, 2)      AS Discount,
  YEAR(o.Order_Date)        AS Order_Year,
  MONTH(o.Order_Date)       AS Order_Month,
  COALESCE(r.Returned, 'No') AS Returned,
  CASE
    WHEN o.Profit > 0 THEN 'Profitable'
    ELSE 'Loss'
  END AS Profit_Flag

FROM [Order] o
LEFT JOIN [Return]  r ON o.Order_ID = r.Order_ID
LEFT JOIN  People   p ON o.Region   = p.Region;

03

Power BI Modeling

Loaded the cleaned SQL query result into Power BI Desktop and set up the data model:

  • Established relationships between Order, Customer, and Product tables
  • Created a separate Date table for time-intelligence functions
  • Defined calculated columns for segmentation and grouping
  • Built all key measures using DAX (see below)
04

DAX Measures

Created custom DAX measures to power the KPIs, growth indicators, and profit margin calculations across both dashboard pages.

DAX — Total Sales & Profit 
Total Sales =
SUM(Query1[Sales])

Total Profit =
SUM(Query1[Profit])

Total Discount =
SUM(Query1[Discount])

Profit Margin % =
DIVIDE(
    SUM(Query1[Profit]),
    SUM(Query1[Sales]),
    0
) * 100
DAX — Year-over-Year Sales Growth 
Target Sales (YoY with Base) =
VAR SelectedYear  = SELECTEDVALUE(Query1[Order_Date].[Year])
VAR BaseTarget    = 400000
VAR PrevYear      = SelectedYear - 1

VAR PrevYearSales =
    CALCULATE(
        SUM(Query1[Sales]),
        FILTER(
            ALL(Query1),
            YEAR(Query1[Order_Date]) = PrevYear
        )
    )

VAR Target =
    IF(
        ISBLANK(PrevYearSales),
        BaseTarget,
        PrevYearSales * 1.10
    )

RETURN Target
DAX — Yearly Profit Margin Goal 
Yearly Profit Margin % =
VAR CurrentMargin =
    DIVIDE(
        CALCULATE(SUM(Query1[Profit])),
        CALCULATE(SUM(Query1[Sales])),
        0
    ) * 100

VAR GoalMargin = 12.00

RETURN
    IF(
        CurrentMargin >= GoalMargin,
        CurrentMargin,
        CurrentMargin
    )
DAX — Customer-Level Metrics 
Total Orders =
DISTINCTCOUNT(Query1[Order_ID])

Total Items =
SUM(Query1[Quantity])

Total Order Amount =
CALCULATE(
    SUM(Query1[Sales]),
    ALLEXCEPT(Query1, Query1[Customer_ID])
)
05

Dashboard Design

Designed two connected dashboard pages in Power BI with a consistent blue-white theme:

  • Sales Summary Page: Overall KPIs (Total Sales $2.30M, Total Profit $286.82K), YoY growth indicator, top 5 customers bar chart, sales by region donut chart, top 10 products table with profit margin, top 10 cities clustered bar chart
  • Customer Insights Page: Customer-level selector with dynamic profile card, sales by year area chart, top 3 categories bar chart, product purchase history table, profit by year waterfall chart, geographic map of customer cities
  • Navigation panel with slicers for Year, Region Manager, Category, Sub Category, and Returns filter
  • Cross-page navigation button: "Customer Insights" ↔ "Sales Data"
06

Challenges & Solutions

●

DAX Context Filtering

Profit margin and growth rate measures gave incorrect results due to filter context issues. Resolved using CALCULATE, ALL(), and FILTER() to explicitly control evaluation context.

●

Performance Optimization

Report was slow with large table visuals. Fixed by reducing unnecessary column imports, optimizing SQL query to load only required fields, and minimizing redundant visuals per page.

●

Filter Conflicts

Slicers on one page were affecting unintended visuals. Managed using Power BI's Edit Interactions panel and Sync Slicers to control exactly which visuals respond to each filter.
